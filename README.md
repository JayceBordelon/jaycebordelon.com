# jaycebordelon.com

Personal portfolio and blog. Plain HTML + Tailwind CSS, generated by a 200-line Node build script. No React, no JSX runtime, no framework. Final image is nginx serving static files.

## Related repos

- [vibetradez.com](https://github.com/JayceBordelon/vibetradez.com) — AI-powered options trading service. Separate stack on a separate droplet; no shared infra.

## What's here

```
src/
├── pages/         Plain HTML page bodies with YAML frontmatter at the top
│   ├── index.html        Hero
│   └── blog/index.html   Blog index (cards generated at build time)
├── partials/      Reused HTML fragments injected by the build
│   ├── layout.html       <head> + body shell with {{ placeholders }}
│   ├── header-home.html  Transparent header used on /
│   ├── header-blog.html  Solid header used on /blog and post pages
│   ├── background.html   Background SVG with 160 animated paths
│   └── post.html         Per-post article wrapper
├── posts/         Blog posts in markdown with YAML frontmatter
├── scripts/       Vanilla JS shipped to the browser (theme toggle,
│                  typing animation, blog filter)
└── styles.css     Tailwind v4 input — design tokens + reveal/glow keyframes

scripts/
└── build.mjs      The whole build. Renders pages, posts, sitemap, robots.

Dockerfile        Builder stage runs `npm run build`, runner is nginx:alpine.
nginx.conf        Pretty URLs, gzip, cache headers, /healthz endpoint.
docker-compose.yml Self-contained stack: Traefik + portfolio container.
```

## Tech stack

| Component | Choice |
|---|---|
| Markup | Plain HTML in `src/pages/` + `src/partials/` |
| Styles | Tailwind v4 (CLI compile), with `globals.css` design tokens copied from the previous shadcn config so the look is identical |
| Content | Markdown in `src/posts/`, rendered via `marked` at build time |
| Interactivity | ~120 LOC of vanilla JS across three files (theme, typing, blog filter). No bundler. |
| Build | One Node script. ~200ms HTML, ~150ms Tailwind. |
| Container | `nginx:alpine`, ~30MB. No Node runtime in production. |

## Local development

```bash
npm install                    # one time, ~5s
npm run build                  # generates dist/, ~500ms total
npm run serve                  # http://localhost:3000

# Live editing: rebuild on save (rerun manually)
npm run watch:css              # background Tailwind watcher
node scripts/build.mjs         # re-emit HTML on demand
```

## Building

```bash
npm run build
```

Runs `node scripts/build.mjs` then the Tailwind CLI. Output is `dist/`. The CI workflow on every PR runs the same command and asserts the expected files exist; any silent build regression that drops a post HTML or the stylesheet fails the PR.

## Adding a blog post

1. Create `src/posts/my-new-post.md` with frontmatter:

```yaml
---
title: "Post title"
summary: "One-line description that lands in OG cards and the index card."
label: "Tutorial"            # rendered as a small badge above the title
author: "Jayce Bordelon"
authorDesc: "SWE"
published: "2026-05-21"      # ISO YYYY-MM-DD; drives sort order
image: "/images/my-cover.gif" # 16:9 cover image (lives in public/images/)
readTime: "8 min read"
tags: ["AI", "Trading"]
---

# Markdown body goes here
```

2. Drop the cover image in `public/images/`.
3. Run `npm run build`. The post appears at `/blog/posts/my-new-post`, on the blog index, and in `sitemap.xml`.

## Hostname routing

Production binds the canonical apex + `www` + the legacy `jayceb.com` permanent redirect. All three resolve to one container. Traefik labels in `docker-compose.yml` declare:

- `jaycebordelon.com` / `www.jaycebordelon.com` → this container, port 3000
- `jayceb.com` / `www.jayceb.com` → 301 redirect to `jaycebordelon.com`

Traefik runs as a sibling service inside this repo's compose with its own letsencrypt cert volume and bridge network. The stack is fully self-contained; deploying to a fresh droplet is `docker compose up -d --build` after DNS is pointed.

## Design system

Carried over from the previous shadcn config: same CSS variables for the palette, same font stack (Plus Jakarta Sans, JetBrains Mono, Source Serif 4 for blog typography), same `--radius` and shadow scale. The shadcn React components themselves are gone; what survives is the design tokens and the Tailwind utility classes, applied directly to plain HTML.

## CI / CD

`.github/workflows/pr-checks.yml` runs on every PR:

1. `npm ci`
2. `npm run build` (HTML + Tailwind)
3. Asserts the expected `dist/` files exist
4. actionlint on the workflow files themselves

Production deploy lives in the operator's deploy pipeline.
